# 完成的定义（Definition of Done）

**源代码质量**
------------------

* 所有变更都已提交到单一代码库（mono-repo）的 `trunk` 或 `main` 分支。
* 代码、测试和文档中不存在任何 lint 错误。
* 所有代码变更至少经过一名其他开发者的评审。
* 通过结对/集体（pair/mob）编程产生的代码视为已评审。
* 所有代码、测试和文档均使用英文编写。

**模块完整性**
---------------------

* 编译期和运行期的警告一律视为错误处理。
* 所有代码均具备完整且细粒度的小型测试。
* 通过解决或合理说明所有代码异味来保持代码质量，包括：
  * 消除重复代码。
  * 将代码元素精简到实现预期功能所需的最小集合。
  * 使用领域语言清晰表达设计意图。
  * 无论测试覆盖情况如何，移除所有未使用的代码。
  * 在必要处添加注释以提高清晰度。

**系统集成**
----------------------

* 不存在已知缺陷；所有缺陷均以最高优先级修复。
* 完成自动化端到端测试，并集成所有内部服务。
* 执行探索式测试，以确保系统的可用性与可靠性。
* 所有外部依赖均保持最新，并已对照最新稳定版本完成验证。
* CI 系统能够成功处理所有变更。
* CI 系统构建在 10 分钟内完成。

**部署与用户体验**
----------------------------------

* 每次 CI 构建成功后，自动部署到生产环境。
* 部署过程中停机时间少于 10 秒 \*
* 始终确保用户数据的完整性与安全性。
* 用户手册保持最新，并及时向用户告知变更内容。
* 已获得产品负责人的批准，但该批准不应阻碍部署。
